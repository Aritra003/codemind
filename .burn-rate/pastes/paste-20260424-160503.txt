## PHASE 1 — MULTI-REPO ARCHITECTURE TESTS [ALL FREE]

We need to know CodeMind works across different TypeScript/JS architectures, not just the two repos we've tested. Clone all five repos first, then test sequentially.

```bash
# Clone all test repos (shallow clone, save disk)
git clone --depth 30 https://github.com/fastify/fastify.git ~/test-repos/fastify
git clone --depth 30 https://github.com/trpc/trpc.git ~/test-repos/trpc
git clone --depth 30 https://github.com/lobehub/lobe-chat.git ~/test-repos/lobe-chat
git clone --depth 30 https://github.com/jaredhanson/passport.git ~/test-repos/passport
git clone --depth 30 https://github.com/colinhacks/zod.git ~/test-repos/zod
```

Why these five:
- **fastify** — plugin-based architecture, heavy use of generics and type inference, `.js` + `.ts` mixed
- **trpc** — pnpm monorepo, deep TypeScript generics, workspace cross-references
- **lobe-chat** — large Next.js app (1000+ files), pnpm monorepo, i18n, AI integrations
- **passport** — old-school CommonJS, `.js` only (no TypeScript), `require()` instead of `import`
- **zod** — single package, pure TypeScript, heavy class inheritance chains, method chaining patterns
